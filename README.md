# WhatsApp Market Snapshot Bot

A Python bot that listens for `/report` in a WhatsApp chat and replies with a Bloomberg-style PDF of the top 10 global stock markets.

**Author:** Hugo Alberto Justisoesetya

---

## Project Overview

| Component | Technology |
|---|---|
| Live market data | yfinance |
| PDF generation | ReportLab + Matplotlib |
| WhatsApp automation | Playwright (Chromium, WhatsApp Web) |
| Configuration | python-dotenv |

The bot targets the **top 10 exchanges by market capitalisation** (Wikipedia / World Federation of Exchanges):
NYSE, Nasdaq, Shanghai SE, Euronext, Japan Exchange, Shenzhen SE, HKEX, NSE India, London SE, Saudi Tadawul.

## Setup

### 1. Install Python dependencies

```bash
pip install -r requirements.txt
```

### 2. Install Playwright browser

```bash
playwright install chromium
```

### 3. Configure environment

```bash
cp .env.example .env
# Edit .env if needed — defaults point to the test number already set
```

### 4. First-time QR scan

On the very first run, a Chromium window opens showing WhatsApp Web's QR code.
Scan it with your phone (**WhatsApp → Linked Devices → Link a Device**).
The session is saved in `./wa_session/` — no re-scan needed afterwards.

## How to Run

### Bot mode (long-running, polls for commands)

```bash
python main.py
```

Send one of these messages in the configured WhatsApp chat:

| Command | Action |
|---|---|
| `/report` | Fetch live data → generate PDF → send as attachment |
| `/quick` | Fetch live data → send plain-text summary |
| `/help` | List available commands |

### Send one report and exit

```bash
python main.py --once
```

Fetches data, generates the PDF, sends it to `WA_RECIPIENT`, then exits.

### Dry run (PDF only, no WhatsApp)

```bash
python main.py --dry-run
```

Generates the PDF and opens it locally — no browser, no WhatsApp.
Useful for iterating on the PDF layout.

## Limitations

- **yfinance is unofficial.** Yahoo Finance can change their API without notice, which may break data fetching.
- **WhatsApp Web automation can break.** WhatsApp updates its web interface regularly; CSS selectors in `whatsapp.py` may need updating.
- **No rate-limiting protection.** Sending `/report` repeatedly in quick succession will spam both yfinance and WhatsApp.
- **Market open/close detection is timezone-based only** — public holidays are not accounted for.
- **Session security** — `wa_session/` contains browser cookies equivalent to a WhatsApp login. Keep it private and exclude it from version control (already in `.gitignore`).
